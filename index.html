Introduction

In-app messaging (IAM) allows developers to send, receive, update and delete messages (SMS and MMS) on behalf of their end user with their explicit consent. The developer application can also use an open (WebSockets / STOMP) or HTTP (WebHooks) commmunication channel to receive real-time communications. Messages can be sent to another phone number (cross carrier), short codes or even to an email address.

API Quick Start
Join Us and Get Your Application Key & Secret

The first step in working with AT&T APIs is to create a developer account or Sign-In to the AT&T Developer Portal.

Click on “My Apps” and Setup new application by providing Application Name and choose In-App Messaging APIs "In-App Messaging-IMMN” or/and "In-App Messaging-MIM" as well as other scopes and submit to get application credentials.

#!/bin/sh

# Obtain these from your API Matrix account
APP_KEY=""
APP_SECRET=""

# Setup the scopes for requesting access 
SCOPE="IMMN,MIM"

# Setup the redirect url for consent flow.
# Note that this must be escaped
REDIRECT_URI="http://localhost"

# Step 1 : Request subscriber authorization : Get end user authorization code
echo "https://api.att.com/oauth/v4/authorize?client_id=${APP_KEY}&scope=${SCOPE}&redirect_uri=${REDIRECT_URI}"

# Input the obtained authorization code obtained from consent.
OAUTH_AUTHORIZATION_CODE="ENTER_VALUE"

# Setup the payload
DATA="grant_type=authorization_code&client_id=${APP_KEY}&client_secret=${APP_SECRET}&code=${OAUTH_AUTHORIZATION_CODE}"

# 2: Exchange code to get access token : Get AccessToken based on oauthorization code
curl "https://api.att.com/oauth/v4/token" \
    --insecure \
    --header "Content-Type: application/x-www-form-urlencoded" \
    --header "Accept: application/json" \
    --data ${DATA}

// can be found at: https://github.com/attdevsupport/codekit-java

import com.att.api.oauth.OAuthService;
import com.att.api.oauth.OAuthToken;

public class OAuthAuthCode {
    public static void main(String[] args) {
        // Enter the value from the 'App Key' field obtained at developer.att.com 
        // in your app account.
        final String clientId = "ENTER VALUE!";

        // Enter the value from the 'App Secret' field obtained at developer.att.com 
        // in your app account.
        final String clientSecret = "ENTER VALUE!";

        // Setup the scopes for requesting access 
        final String scope = "IMMN,MIM";

        // Setup the redirect url for consent flow.
        // Note that this must be escaped
        final String redirectUri = "http://localhost";

        // Step 1: Request subscriber authorization : Get end user authorization code
        final String consent = "https://api.att.com/oauth/v4/authorize?client_id="
            + clientId + "&scope=" + scope + "&redirect_uri=" + redirectUri;

        System.out.println("Please authenticate using the following url: " + consent);

        final String authCode = "ENTER_VALUE";
        // Create a service for requesting an OAuth access token.
        OAuthService osrvc = new OAuthService("https://api.att.com", clientId, clientSecret);

        // Step 2: Obtain the token using the authorization code
        OAuthToken token = osrvc.getTokenUsingCode(authCode);
        System.out.println("Access token: " + token.getAccessToken());
    }
}

API Reference
Create Message Index
Overview

The Create Message Index method allows the developer to create an index cache for the user's AT&T Message inbox with prior consent.

  • OAuth Scope

  • Scope: MIM

  • Model: auth_code

  • Resource
  • copy 
    POST https://api.att.com/myMessages/v2/messages/index
Examples

Request

copy 
POST /myMessages/v2/messages/index  HTTP/1.1
host: api.att.com
authorization: Bearer xyz123456789
accept: application/json

Response

copy 
HTTP/1.1 202 Accepted
Request Parameters
Parameter
accept String Header
Specifies the format of the body for the response. The acceptable values for this parameter are:
  • application/json
  • application/xml
The default value is application/json. Note: For this method, this parameter specifies how the entity should be represented in case of an error. This parameter is for setting the format of an error message. If there is no error, then the representation will match the form of the actual content.
authorization String Header
Specifies the authorization. The acceptable format for this parameter is the word "Bearer" followed by a space ( ) and an OAuth access token. If this parameter value is missing from the header, then the API Gateway returns a message with an HTTP Status Code of 400 Invalid Request. If the OAuth access token is not valid, then the API Gateway returns an HTTP Status Code of 401 Unauthorized with a WWW-Authenticate HTTP header.
Response Parameters
Parameter
N/A N/A Not applicable
Specifies that no parameters are returned.

Response Parameters
Parameter
message Message Object Body
Specifies the message details and the related meta-data.

Specifies the unique identifier for the message. This is the child parameter of the Message object.
from Object Body
Contains the sender for the message. This is the child parameter of the Message object.
recipients List of Value Objects Body
Contains the parties involved in the message. For outgoing messages, this parameter includes the phone numbers of all of the parties. For incoming messages, this parameter includes the phone numbers of all of the parties, except for the user's phone number. If the message is a group message, then this parameter may contain multiple addresses. This is the child object of the Message object.
value String Body
Specifies the address of the originator or the recipient. This is the child parameter of the from or recipient parameter.
text String Body
Specifies the text portion of the message. Note: If the Type parameter is set to MMS, then this parameter is not present or empty. This is the child parameter of the Message object.
mmsContent list of Objects Body
Contains the multi-media message parts (attachments) of the message. Note: If the Type parameter is set to TEXT, then this parameter is not present or empty. This is the child object of the Message object.
contentName String Body
Specifies the content name for the given message part or attachment. This is a child parameter of the MMSContent object.
type String Body
Specifies the multi-media type of the message part or attachment. The acceptable values for this parameter are:
  • IMAGE
  • VIDEO
  • TEXT
  • AUDIO
This is a child parameter of the MMSContent object.
contentType String Body
Specifies the content type identifier for the message part or attachment. Example: image/jpeg. This is a child parameter of the MMSContent object.
contentUrl String Body
Specifies the resource URI path to retrieve the given message part or attachment. This is a child parameter of the MMSContent object.
timeStamp DateTime Body
Specifies the date and time that the message was stored in the AT&T Message inbox. Note: This parameter is specified using Coordinated Universal Time (UTC), formerly referenced as the Greenwich Mean Time (GMT) time zone.
isUnread Boolean Body
Indicates that the message has not been read. This is the child parameter of the Message object.
isIncoming Boolean Body
Indicates that the message is an incoming message and not an outgoing message. This is the child parameter of the Message object.
type String Body
Specifies the message type, either TEXT or MMS. The acceptable values for this parameter are:
  • TEXT : Used for SMS messages.
  • MMS
typeMetaData MessageMetaData Object Body
Contains the message-specific meta-data fields. This is the child parameter of the Message object.
subject String Body
Specifies the subject for the message. Note: This is an optional parameter and is only applicable when the Type parameter is set to MMS. This is a child parameter of the typeMetaData object.
isSegmented Boolean Body
Indicates that the message is segmented into multiple parts, for SMS text messages. This is the child parameter of the typeMetaData object.
segmentationDetails SegmentationDetails Object Body
Contains the details of the segmentation of the SMS message. This is the child parameter of the typeMetaData object.
segmentationMsgRefNumber Integer Body
Specifies the reference number for messages that are part of the same segmented SMS message. This is a child parameter of the SegmentationDetails object.
totalNumberOfParts Integer Body
Specifies the total number of segments for this segmented SMS message. This is a child parameter of the SegmentationDetails object.
thisPartNumber Integer Body
Specifies the segmentation part number for this particular message. This is a child parameter of the SegmenationDetails object.

If the OAuth access token is not valid, then the API Gateway returns an HTTP Status Code of 401 Unauthorized with a WWW-Authenticate HTTP header.
messageIds String QueryString
Specifies a comma-separated list of messageIds for the messages that must be retrieved from the AT&T Message inbox. If this parameter is specified in the request, then this parameter value takes preference over any other parameters specified in the request and in fact the Messaging system ignores any other parameters specified in the requests. If all of the message identifiers specified are not found, then a response containing an error message is returned. If one or more, but not all, of the message idenifiers specified are not found, then the message identifiers that are not found are listed in the failedMessages parameter in the a successful response.

Specifies the upper limit of messages that would be returned in the response. The acceptable values for this parameter are:
  • 1 through 500

Specifies the offset from the beginning of the ordered set of messages from where requested filters shall be applied.
isUnread Boolean QueryString
Indicates that the returned set of messages are filtered based on whether each message is marked as unread or not.
isIncoming Boolean QueryString
Indicates that the returned set of messages are filtered based on whether each message is inbound or outbound.

Specifies a comma-separated list of the types of the messages that are returned in the response.

Specifies the message receipient address as a filter criteria so that the messages that have receipients containing or matching the provided receipient address are returned. Currently this parameter supports only a single phone number. The number specified should be the sender in an incoming message or the recipient in an outgoing message. Note: When the mobile number is passed as the search keyword, the prefix of +1 should not be included for North American numbers.

Get Messages Delta
Overview

The Get Messages Delta method provides the capability to check for updates by passing in a client state. This is typically used when an app changes from being offline to becoming online. Get Notification Connection Details
Overview

The Get Notification Connection Details method retrieves details about the credentials, endpoint, and resource information to required to set up a notification connection. This method needs to be invoked before a Simple Text Oriented Messaging Protocol (STOMP) connection is established to subscribe for notifications. The user credentials provided in the response are valid for 30 seconds and a STOMP connection needs to be established within this time period. If the credentials expire, then this method must be requested again. After the developer has received the connection details, the developer must use that information to set up an out-of-band connection to the notification server. This connection does not traverse the API Gateway and all of the actual connection setup is handled between the developer and the notification server. This method only retrieves the information needed to set up the connection. Details about the notification connection setup are described in a .pdf file. Note: The developer should ensure that the message index cache for the app is initialized before setting up a notification channel.

  • OAuth Scope

  • Scope: MIM

  • Model: auth_code

Send Message
Overview

The Send Message method sends messages on behalf of an AT&T Wireless user. Each time this method request is sent, a single message may be sent to one or more destination mobile devices. Messages are processed synchronously and sent asynchronously to the destination mobile device. The request may have up to 21 attachments, but the total size of the message must be 1 megabyte (MB) or less. There are two ways that an MMS message can be sent: by using a MIME-encoded message, or by embedding the Base64 encoded representation of the media in the JSON or XML element.

  • This method sends messages as SMS or MMS as follows.
    • For SMS Messages: Text only, using the ISO-8859-1 format. The acceptable maximum length is 160 characters.
    • For MMS messages: Any attachment, or text with a length greater than 160 characters.
  • This method supports the following MMS content types.
    • application/vnd.wap.multipart.mixed - without SMIL component.
    • application/vnd.wap.multipart.related - without SMIL component.
    • application/vnd.wap.multipart.related - with SMIL component.
  • This method is limited to the following content types for MMS messages.
    • ASCII
    • UTF-8
    • UTF-16
    • ISO-8859-1
    • JPEG
    • GIF87a
    • GIF89a
    • 3GPP
    • MP4
    • AMR-NB
    • SP-MIDI
    • MP3
    • AAC-LC
    • V-card PIM
    • PIM Support - vCalendar 1.0

  • Scope: IMMN

  • Model: auth_code

Update Message
Overview

The Update Message method gives the developer the ability to update some flags related to one of the messages in a user's AT&T Message inbox with prior consent.

  • OAuth Scope

  • Scope: MIM

  • Model: auth_code

  • Resource
  • copy 
    PUT https://api.att.com/myMessages/v2/messages/{{messageId}
Examples
  • JSON
  • XML

Request

copy 
PUT /myMessages/v2/messages/a123  HTTP/1.1
Host: api.att.com
authorization: Bearer xyz123456789
accept: application/json
content-type: application/json
Content-length: 49

{
    "message": {
        "isUnread":true
    }
}

Response

copy 
HTTP/1.1 204 No Content

Request

copy 
PUT /myMessages/v2/messages/a123 HTTP/1.1
Host: api.att.com
authorization: Bearer xyz123456789
accept: application/xml
content-type: application/xml
Content-length: 49

<message>
    <isUnread>true</isUnread>
</message>

Response

copy 
{
    "RequestError":{
        "PolicyException":{
            "MessageId":"SVC0001"
            "Text":"The following message could not be updated: a123"
        }
    }
}

  • Model: auth_code

  • copy

  • XML

copy

authorization: Bearer xyz123456789
accept: application/json
content-type: application/json

    "messages": [
        {
            "messageId":"a123",
            "isUnread":true
        },

            "messageId":"b456",
            "isUnread":true

 }
