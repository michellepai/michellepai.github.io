Introduction

The In-App Messaging API sends and retrieves SMS and MMS messages on behalf of a customer after consent. The message is sent to the recipients using the customer's phone number as the sender address. A develoepr can build apps to send and manage messages from single application.

  • Sending messages from your app: The Send Messages (text and images) to any mobile phone number, emails and shortcodes
  • Receiving messages in your app: Responses to the messages are able to be received by the user's mobile phone

OAuth

The APIs provided by AT&T use the OAuth 2.0 framework, and an OAuth access token must be included in each request to the API Gateway. To obtain an OAuth access token, use the App Key and App Secret from the My Apps section after registering and signing into your API Platform account.

You may authorize access for this API by using the following scopes.

"IMMN" for the Send Message method.

"MIM" for the Create Message Index method, the Delete Message method, the Delete Messages method, the Get Message Content method, the Get Message Index Info method, the Get Message List method, the Get Message method, the Get Messages Delta method, the Get Notification Connection Details method, the Update Message method, and the Update Messages method.

Authorization Code

To obtain user consent authorization run the following command from the Web browser and replace "{{APP_KEY}}" and "{{API_SCOPES}}" with the App Key and the API scopes that you wish to access as follows.

Get User Authorization method using the HTTP request:

copy 
https://api.att.com/oauth/authorize?client_id={{APP_KEY}}&scope={{API_SCOPES}}&redirect_uri={{REDIRECT_URI}}

A successful response from the API Gateway opens the User Consent Authorization page. After the user has completed the User Consent Authorization page, the API Gateway sends a response. Extract the value from "{{OAUTH_AUTHORIZATION_CODE}}" to use in the Get Access Token request to the API Gateway.

Response URI for successful OAuth Authorization Code:

https://{{YOUR_OAUTH_URI}}?code={{OAUTH_AUTHORIZATION_CODE}}

After obtaining the OAuth authorization code run the following command from the Command Line Interface window and replace "{{APP_KEY}}", "{{APP_SECRET}}", and "{{OAUTH_AUTHORIZATION_CODE}}" with the App Key, App Secret, and the OAuth authorization code as follows.

Get Notification Connection Details
Overview

The Get Notification Connection Details method retrieves details about the credentials, endpoint, and resource information to required to set up a notification connection. This method needs to be invoked before a Simple Text Oriented Messaging Protocol (STOMP) connection is established to subscribe for notifications. The user credentials provided in the response are valid for 30 seconds and a STOMP connection needs to be established within this time period. If the credentials expire, then this method must be requested again. After the developer has received the connection details, the developer must use that information to set up an out-of-band connection to the notification server. This connection does not traverse the API Gateway and all of the actual connection setup is handled between the developer and the notification server. This method only retrieves the information needed to set up the connection. Details about the notification connection setup are described in a .pdf file. Note: The developer should ensure that the message index cache for the app is initialized before setting up a notification channel.

Send Message
Overview

The Send Message method sends messages on behalf of an AT&T Wireless user. Each time this method request is sent, a single message may be sent to one or more destination mobile devices. Messages are processed synchronously and sent asynchronously to the destination mobile device. The request may have up to 21 attachments, but the total size of the message must be 1 megabyte (MB) or less. There are two ways that an MMS message can be sent: by using a MIME-encoded message, or by embedding the Base64 encoded representation of the media in the JSON or XML element.

  • This method sends messages as SMS or MMS as follows.
    • For SMS Messages: Text only, using the ISO-8859-1 format. The acceptable maximum length is 160 characters.
    • For MMS messages: Any attachment, or text with a length greater than 160 characters.
  • This method supports the following MMS content types.
    • application/vnd.wap.multipart.mixed - without SMIL component.
    • application/vnd.wap.multipart.related - without SMIL component.
    • application/vnd.wap.multipart.related - with SMIL component.
  • This method is limited to the following content types for MMS messages.
    • ASCII
    • UTF-8
    • UTF-16
    • ISO-8859-1
    • JPEG
    • GIF87a
    • GIF89a
    • 3GPP
    • MP4
    • AMR-NB
    • SP-MIDI
    • MP3
    • AAC-LC
    • V-card PIM
    • PIM Support - vCalendar 1.0
